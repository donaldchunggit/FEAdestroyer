# FEA Destroyer 🏗️

> **A Physics-Informed Graph Neural Network surrogate for cantilever beam structural analysis**

## What This Is

A GNN-based surrogate model that learns to predict nodal displacements and Von Mises stress fields for cantilever beams under arbitrary transverse loading — trained on real finite element data, deployed through an interactive web UI.

The model replaces a full FEM solve at inference time. Once trained it predicts in milliseconds. Training data is generated by a real sparse FEM solver (linear-elastic tetrahedral elements), so there is no artificial noise ceiling from analytical approximations.

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt
pip install torch torchvision torchaudio            # CPU
# or for NVIDIA GPU:
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install torch-geometric

# 2. Generate data + train in one command
python run_fresh.py

# 3. Launch the frontend
streamlit run app.py
```

`run_fresh.py` automatically detects your hardware and adjusts everything accordingly — no manual config needed.

## Dataset Generation

Two generators are available:

### FEM Generator (default) — `generator_fem/fem_generator.py`
- Runs a real sparse linear-elastic FEM solve (KU = F) for each sample
- Tetrahedral mesh via scipy Delaunay on a perturbed structured grid
- Degenerate element filtering before the solve (~25% element reduction)
- **Multiprocessed** — uses all available CPU cores in parallel
- 6 materials × random geometry × random force = true physics ground truth
- No noise floor: can achieve Pearson R > 0.95

### Analytical Generator (fast) — `generator_adv/adv_generator.py`
- Beam theory formulas with ±10% added noise
- Instant generation, but accuracy ceiling ~R = 0.85
- Use with `python run_fresh.py --generator fast`

## Model Architecture

**EngineeringGNN** — `models/engineering_gnn.py`

- Node encoder: Linear(12 → 256) with LayerNorm
- Edge encoder: Linear(6 → 256) with LayerNorm
- 5× GINEConv message-passing layers with residual connections
- Displacement head → per-graph RMS normalisation → learned scale multiplier
- Stress head → log-space prediction → exp (prevents negative stress)
- Safety factor: σ_yield / σ_von_mises per node

Node features (12 dims): coordinates, fixed mask, distance-to-root, tip flag, force direction, log(|F|), log(E), ν

## Training

Curriculum learning over 4 stages:

| Stage | Epochs (GPU) | Focus |
|---|---|---|
| Warmup | 75 | Scale initialisation |
| Scale | 125 | Displacement magnitude |
| Correlation | 400 | Spatial pattern learning |
| Stress | 400 | Full multi-task training |

Best model is selected by **Pearson R on displacement** (not loss), saved to `best_model_correlation.pt`.

Early stopping triggers at R > 0.70.

## Deployment Standards

The training script prints a deployment assessment at the end of every run:

| R (Pearson, displacement) | Status |
|---|---|
| ≥ 0.99 | Production surrogate — can replace FEA in design loops |
| ≥ 0.95 | Deployment ready — preliminary design and trend analysis |
| ≥ 0.90 | Screening only — every result needs FEA validation |
| < 0.90 | Not deployable — results cannot be trusted |

## Web Frontend

```bash
streamlit run app.py
```

The interactive UI lets you:
- Set beam geometry (length, width, height via sliders)
- Choose material (6 structural materials with real E, ν, σ_yield)
- Apply a transverse force with magnitude and direction angle
- View a 3D deformed mesh coloured by Von Mises stress or displacement magnitude
- See max displacement (mm), max stress (MPa), and safety factor
- Compare against closed-form beam theory values for sanity checking

The frontend displays the model's training R value and automatically warns if predictions are not reliable.

## Installation Notes

PyTorch and PyG must be installed manually to match your CUDA version:

```bash
# CPU only
pip install torch torchvision torchaudio
pip install torch-geometric

# NVIDIA GPU (CUDA 12.1)
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install torch-geometric
pip install torch-scatter torch-sparse torch-cluster torch-spline-conv \
    -f https://data.pyg.org/whl/torch-<VERSION>+cu121.html
```

Everything else installs via:
```bash
pip install -r requirements.txt
```

## Limitations

- Geometry is limited to solid rectangular cantilever beams (fixed at one end, loaded at the other)
- The FEM solver uses linear elasticity — no plasticity, buckling, or contact
- Model accuracy depends on training data quality and quantity; always validate critical designs with certified FEA software
- Not a substitute for professional structural engineering judgement
